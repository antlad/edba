Class statement

edba::statement — This class represents a prepared (or ordinary) statement that can be executed.

Description

This object is usually created via session::prepare() function.

statement public construct/copy/destruct

  1. statement();

    Default constructor, provided for convenience, access to some member function of empty statement will cause an exception empty_statement being thrown. 

  2. statement(const backend::connection_ptr & conn, 
          const backend::statement_ptr & stmt);

statement friend functions

  1. friend bool operator==(const statement & s1, const statement & s2);
    Equality operator.

statement public member functions

  1. statement & reset_bindings();

    Reset the statement - remove all bindings and return it into initial state so query() or exec() functions can be called once again.

    You must use it if you use the same statement multiple times.

    Immediatelly exits for empty statements 

  2. template<typename T> statement & bind(int col, const T & v);

    Bind a value v to the placeholder by index (starting from the 1).

    Placeholders are marked as ':placeholdername' in the query. If placeholder index is higher then the number placeholders is the statement it may throw invalid_column exception.

    If placeholder was not binded the behavior is undefined and may vary between different backends.

    Immediatelly exits for empty statements 

  3. statement & bind(int col, const bind_types_variant & v);

    Bind a value v to the placeholder by index (starting from the 1).

    Placeholders are marked as ':placeholdername' in the query. If placeholder index is higher then the number placeholders is the statement it may throw invalid_column exception.

    If placeholder was not binded the behavior is undefined and may vary between different backends.

    Immediatelly exits for empty statements 

  4. template<typename T> statement & bind(const string_ref & name, const T & v);

    Bind a value v to the placeholder by name.

    Placeholders are marked as ':placeholdername' in the query. If placeholder name is invalid then it may throw invalid_column exception

    If placeholder was not binded the behavior is undefined and may vary between different backends.

    Immediatelly exits for empty statements 

  5. statement & bind(const string_ref & name, const bind_types_variant & v);

    Bind a value v to the placeholder by name.

    Placeholders are marked as ':placeholdername' in the query. If placeholder name is invalid then it may throw invalid_column exception

    If placeholder was not binded the behavior is undefined and may vary between different backends.

    Immediatelly exits for empty statements 

  6. template<typename T> statement & bind(const T & v);

    Bind a value v to the placeholder.

    Placeholders are marked as ':placeholdername' in the query. If number of calls is higher then the number placeholders is the statement it may throw invalid_column exception.

    If placeholder was not binded the behavior is undefined and may vary between different backends.

    Immediatelly exits for empty statements 

  7. long long last_insert_id();

    Get last insert id from the last executed statement, note, it is the same as sequence_last("").

    Some backends requires explicit sequence name so you should use sequence_last("sequence_name") in such case.

    If the statement is actually query, the behavior is undefined and may vary between backends.

    Throw empty_statement exception for empty statements 

  8. long long sequence_last(std::string const & seq);

    Get last created sequence value from the last executed statement.

    If the backend does not support named sequences but rather supports "auto increment" columns (like MySQL, Sqlite3), the seq parameter is ignored.

    If the statement is actually query, the behavior is undefined and may vary between backends.

    Throw empty_statement exception for empty statements 

  9. unsigned long long affected();

    Get the number of affected rows by the last statement,

    If the statement is actually query, the behavior is undefined and may vary between backends.

    For empty statements return 0 

  10. row first_row();

    Fetch a single row from the query.

    If the result set consists of more then one row it throws multiple_rows_query exception, however some backends may ignore this.

    Would throw empty_row_access exception if the result is empty.

    If the statement is not query statement (like SELECT) it would likely throw an exception, however the behavior may vary between backends that may ignore this error.

    Throw empty_statement exception for empty statements 

  11. rowset query();

    Fetch a result of the query, if the statement is not query statement (like SELECT) it would likely throw an exception, however the behavior may vary between backends that may ignore this error.

    Throw empty_statement exception for empty statements 

  12. template<typename T> operator rowset< T >();
    Same as query() - syntactic sugar. 
  13. void exec();

    Execute a statement, of the statement is actually SELECT like operator, it throws edba_error exception, however the behavior may vary between backends that may ignore this error.

    Immediatelly exits for empty statements 

  14. template<typename T> statement & operator<<(const T & v);

    Syntactic sugar for 

    st.bind(val);


  15. statement & operator<<(void(*)(statement &st) manipulator);
    Apply manipulator on the statement, same as manipulator(*this). 
  16. row operator<<(row(*)(statement &st) manipulator);
    Apply manipulator on the statement, same as manipulator(*this). 
  17. rowset operator<<(rowset<>(*)(statement &st) manipulator);
    Apply manipulator on the statement, same as manipulator(*this). 
  18. template<typename T1, typename T2> statement & operator<<(unspecified val);

    Used with types produced by use function.

    The call st << use("paramname", val) is same as

    st.bind("paramname", val);
