Description

Thread-safe pool of sessions with maximum number limit

session_pool public construct/copy/destruct

  1. template<typename Driver> 
  session_pool(Driver driver, const char * conn_string, int max_pool_size, 
               session_monitor * sm = 0);

    Construct pool of session with max limit. All sessions will be created using specified driver and conn_string. Constructor doesn`t create connection itself, instead they will be created lazily by open and try_open calls. 

  2. session_pool(const session_pool &);
  3. session_pool& operator=(const session_pool &);

session_pool public member functions

  1. void invoke_on_connect(const conn_init_callback & callback);

    Invoke provided function object once on connection creation. This allow to setup all sessions in pool in uniform manner. configure call doesn`t affect already created sessions it will be applied only to the new one. 

  2. session open();

    Get session from pool or create new one if there is no free sessions and max_pool_size limit is not exceeded. If there is no free sessions and max_pool_size limit exceeded then wait until someone will release session. 

  3. bool try_open(session & sess);

    Get session from pool or create new one if there is no free sessions and max_pool_size limit is not exceeded. If there is no free sessions and max_pool_size limit exceeded then return false and leave sess untouched 

  4. double total_execution_time();

    Return total time in seconds spent by all session on query and statement execution
